## DMCA Takedown Process

Upon receiving a valid report, the following steps will be taken:

1. **Verification** — The infringing work is reviewed within 48 hours
2. **Notice to infringer** — A cease-and-desist is sent directly if contact info is available
3. **Platform takedown** — A formal DMCA notice is filed with the hosting provider (GitHub, Netlify, Vercel, Cloudflare, etc.)
4. **Escalation** — If unresolved, the matter is escalated through legal channels

## Reporting a Security Vulnerability

This is a static front-end portfolio with no backend, database, or user authentication. There is minimal attack surface, but if you discover a vulnerability (e.g. a malicious third-party script, XSS vector, or supply chain issue):

**Do NOT open a public GitHub Issue.**

Instead, email directly:  
📧 **harshaldonarkar@gmail.com**  
**Subject:** `[SECURITY] Vulnerability report — portfolio`

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Your recommended fix (optional)

You will receive a response within 72 hours. Responsible disclosure is appreciated.
